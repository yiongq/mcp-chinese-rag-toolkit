Hello-world MCP RAG server for Chinese documents. Uses BM25 (jieba FTS5) over
a small sample doc out of the box; swap in the real `bge-large-zh-v1.5`
embedder + hybrid + rerank pipeline when you're ready.

## 1. Install

```sh
pnpm install
```

## 2. Build the index

```sh
pnpm build-index
# → Indexed N chunks → data/index.db
```

## 3. Start the MCP server (stdio transport)

```sh
pnpm start:stdio
```

## 4. Try it via MCP Inspector

In a second terminal:

```sh
npx @modelcontextprotocol/inspector pnpm start:stdio
```

Call `search_docs` with `{ "query": "试用期工资" }` — you should see a chunk
from `sample-doc.md`.

## Switch to a real Chinese embedder

The mock zero-vector embedding makes hello-world cold-start instant. To enable
semantic vector search:

1. Edit `scripts/build-index.ts`:

   ```ts
   import { loadEmbedder, chunk, openIndex } from '@yiong/mcp-chinese-rag-toolkit';

   const embedder = await loadEmbedder(); // downloads ~400 MB on first run
   const vectors = await embedder.embedBatch(chunks.map((c) => c.content));
   const rows = chunks.map((c, i) => ({ chunk: c, embedding: vectors[i]! }));
   ```

2. Edit `src/server.ts` to swap `ftsSearch` for `createHybridSearch(...)`.
   See toolkit README §Hybrid Search + §Reranker for the wiring.
